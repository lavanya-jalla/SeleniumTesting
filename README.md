# Selenium Testing with Java

## Project Overview

This repository contains **automation testing scripts using Selenium WebDriver with Java**.  
The project demonstrates how to automate browser actions, interact with web elements, and perform functional testing on web applications.

Selenium WebDriver is widely used for **automating web applications for testing purposes**, allowing testers to simulate user actions such as clicking buttons, filling forms, handling alerts, and navigating pages.

## Features

- Web automation using Selenium WebDriver
- Handling web elements (buttons, inputs, links)
- XPath and locator strategies
- Browser automation using ChromeDriver
- Example automation test scripts

 ## Technologies Used

- Java

  ## Concepts Used

This project demonstrates different Selenium locator strategies including:
- ID : id is the fastest and most reliable locator in Selenium because IDs are usually unique.
- Name: The name locator finds elements using the name attribute.
- ClassName: This locator finds elements using the class attribute.
- LinkText: Used to locate links using the exact visible text.
- PartialLinkText: Used to locate links using partial text.
- XPath: XPath is one of the most powerful locators used when other locators are not available.
- CSS Selector: CSS selectors locate elements using CSS patterns.
